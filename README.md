## ¿Para qué creen que sirve Git en proyectos reales?

Para poder trabajar conjuntamente, para tener un historial de cambios dentro del proyecto con fines de registro o por si algo se rompe, para que el entorno rigido de la computadora pueda ser mas experimental al poder crear branch's del proyecto y que la informacion dentro del proyecto pueda ser vista por otras personas en cual quier momento para su posterior retroalimentacion manejando la privacidad de manera manual.

## ¿Por qué es importante estructurar bien un proyecto?

Para que este sea colaborativa o sirva de herencia para futuros trabajos. Un proyecto bien estructurado puede ser estudiado para aprendizaje, modificaciones o mejoras de manera mas intuitiva sin problemas de comprension.
